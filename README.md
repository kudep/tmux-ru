# tmux-ru

Готовый конфиг tmux на базе [oh-my-tmux](https://github.com/gpakosz/.tmux) с
человеческими настройками мыши, буфера обмена и зума — и, главное, с
**поддержкой русской раскладки**: все хоткеи продолжают работать, когда
клавиатура переключена на кириллицу.

## Установка в одну команду

```sh
bash -c "$(curl -fsSL https://raw.githubusercontent.com/kudep/tmux-ru/main/install.sh)"
```

Установщик сам поставит зависимости (`tmux`, `xsel`) через `apt`, сделает
резервные копии существующих `~/.tmux.conf` / `~/.tmux.conf.local` и разложит
конфиг. Ничего не затирается без бэкапа.

После установки — открой новый tmux или перезагрузи текущий:

```sh
tmux source-file ~/.tmux.conf
```

## Что внутри

| Возможность | Как пользоваться |
|-------------|------------------|
| **Мышь** | Клик по имени окна в статус-баре — переключение вкладок; колёсико — скролл лога без ручного copy-mode; тянуть границу — ресайз панели. |
| **Буфер обмена** | Выделение мышью и `y`/`Enter` в copy-mode кладут текст прямо в системный буфер X11 (`xsel`). |
| **Зум панели** | `Alt+z` — развернуть панель на всё окно, ещё раз — вернуть. Дублируется `prefix + z`. |
| **История** | Буфер прокрутки 50 000 строк. |
| **Русская раскладка** | Все биндинги работают и в кириллице (см. ниже). |

## Как работает поддержка русской раскладки

Когда раскладка переключена на русскую, физическая клавиша шлёт кириллический
символ (`z → я`, `q → й`, `y → н`, …), и буквенные биндинги tmux перестают
срабатывать. `tmux/cyrillic-layout.sh` читает все биндинги в таблицах `prefix`,
`copy-mode`, `copy-mode-vi`, `root` и создаёт для каждого дубликат под
кириллической клавишей с той же физической позицией (ЙЦУКЕН) — включая
`Alt`-комбинации (`Alt+z → Alt+я`).

Скрипт запускается по хуку `client-attached` (когда появляется клиент — то есть
после того, как oh-my-tmux донастроил все биндинги, в том числе клавиши буфера
обмена) и идемпотентен, поэтому повторные запуски безопасны.

**Ограничения (сознательно):** не зеркалируются
- сдвинутые символы (`%`, `"`, `!` …) — в русской раскладке они на других
  физических клавишах;
- команды с `${...}` (нишевые бинды правки конфига вроде `prefix + e`) — они не
  переживают повторный разбор `source-file`.

Ежедневная навигация, copy-mode, зум и работа с буфером — покрыты полностью.

## Тесты

Полный набор гоняется в контейнере с **чистой Ubuntu** — там же проверяется и
сама установка одной командой:

```sh
test/run.sh          # docker build + docker run
```

Проверяется: установка на чистой системе, загрузка конфига, совпадение
кириллических зеркал с латинскими оригиналами, идемпотентность, и **end-to-end**
— реальные кириллические нажатия подаются через настоящий pty-клиент (потому что
`send-keys` обходит таблицы клавиш tmux), после чего проверяется, что
`Alt+я` зумит панель, `prefix + с` открывает окно, `prefix + х` / `й` входит и
выходит из copy-mode.

## Удаление

```sh
rm -f ~/.tmux.conf ~/.tmux.conf.local ~/.tmux/cyrillic-layout.sh
# восстановить бэкапы, если нужно:
ls -d ~/.tmux.conf*.bak-* ~/.tmux.conf.local.bak-* 2>/dev/null
```
